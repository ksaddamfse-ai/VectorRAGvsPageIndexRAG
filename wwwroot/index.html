Vector RAG vs PageIndex RAG

Benchmark comparing two RAG strategies on a resume PDF. Same LLM backend (OpenRouter), same embedding model.

Vector RAG

PDF → text → chunks → embeddings → Qdrant vector DB

Query: embed question → nearest-neighbor search → LLM answer

Pros: Semantic search finds related content even without keyword match. Fast retrieval (~2–6s).

Cons: Chunk boundaries may split relevant context. No hierarchy awareness.

PageIndex RAG

PDF → text → LLM extracts tree structure → SQLite store

Query: LLM selects relevant tree nodes → return node text → LLM answer

Pros: Hierarchical understanding. Section-aware context.

Cons: Two LLM calls per query (nav + answer). Slower (~3–30s). Tree quality depends on LLM.
